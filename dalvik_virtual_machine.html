Dalvik Virtual Machine

  • Dalvik Virtual Machine (DVM) Android operating system के शुरुआती version में उपयोग की जाने वाली virtual machine थी, जो खासतौर पर mobile devices के लिए design की गई थी.

  • Java का code सीधे DVM पर नहीं चल सकता था, इसलिए इसे पहले Dalvik Executable (.dex) formate में convert किया जाता था, जिससे यह optimized तरीके से run हो सके.

  • DVM में Just-In-Time (JIT) compiler का उपयोग किया जाता था, जिससे runtime पर code को compile कर performance को बेहतर बनाया जाता था.

  • DVM कम memory और बैटरी की खपत करता था क्योंकि यह register-based architecture पर काम करता था, जबकि Java Virtual Machine (JVM) stack-based architecture का उपयोग करती थी और ज्यादा memory लेती थी.

  • DVM multitasking को support करता था, जिससे एक साथ कई apps बिना किसी रुकावट के run हो सकती थीं.

  • DVM, Linux Kernel के ऊपर चलता था, जो memory management, security और process scheduling को control करता था.

  • जब कोई app install होती थी, तो उसका Java code पहले .dex formate में बदला जाता था, फिर DVM इसे runtime पर interpret कर app को execute करता था.

  • Android 5.0 (Lollipop) के बाद DVM को Android Runtime (ART) से replace कर दिया गया, क्योंकि ART Ahead-Of-Time (AOT) compilation का उपयोग करता है, जिससे app पहले से ही optomize होकर install होती है और ज्यादा तेज़ी से run होती है.

  • DVM ने Android app development को बेहतर बनाया, क्योंकि यह कम RAM और CPU power में अच्छा performance, battery saving, और secure app execution प्रदान करता था.

  • हालांकि अब DVM का उपयोग नहीं होता, लेकिन यह Android के शुरुआती development में एक महत्वपूर्ण भूमिका निभाता था.
